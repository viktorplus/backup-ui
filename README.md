# Backup UI

Веб-интерфейс для наблюдения за Linux/Docker сервером, настройки backup-планов и ручного запуска резервного копирования.

Проект рассчитан на запуск как отдельный контейнер рядом с сервисами сервера. По умолчанию UI слушает только `127.0.0.1:8090`, а доступ выполняется через SSH tunnel или reverse proxy с собственной авторизацией.

## Возможности

- анализ проектов в `/opt` и docker-compose каталогов;
- read-only сводка сервера: hostname, IP, CPU, память, диски, load average, процессы;
- read-only просмотр директорий хоста;
- обнаружение PostgreSQL, MySQL и MariaDB контейнеров;
- просмотр баз и таблиц без чтения строк данных;
- сохранение профиля конкретного сервера без секретов;
- создание backup-планов по проектам, конфигам и базам;
- локальное и SSH-хранилище для копий;
- dry-run retention и ручное применение retention;
- отдельный restore mode через compose override.

## Быстрый запуск

```bash
cd /opt/backup-ui
docker compose up -d --build
```

Подключение:

```bash
ssh -L 8090:127.0.0.1:8090 root@SERVER
```

Открыть:

```text
http://127.0.0.1:8090
```

## Безопасность

В репозиторий не должны попадать:

- реальные backup-архивы;
- приватные ключи;
- пароли баз данных;
- `.env` с секретами;
- state DB приложения;
- серверные отчеты и профили с приватными заметками.

UI использует Docker socket и read-only mount корня хоста для анализа. Поэтому публиковать UI напрямую в интернет нельзя.
