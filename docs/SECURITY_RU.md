# Безопасность

Backup UI предназначен для закрытого административного доступа.

## Не Публиковать Напрямую

Контейнер имеет доступ к:

- Docker socket;
- read-only корню хоста;
- `/opt/backups` на запись;
- Docker CLI хоста.

Это удобно для backup и анализа, но требует строгого ограничения доступа. Рекомендуемый доступ:

```bash
ssh -L 8090:127.0.0.1:8090 root@SERVER
```

Или reverse proxy с отдельной авторизацией, TLS и ограничением IP.

## Секреты

В git нельзя добавлять:

- `.env` с паролями;
- приватные SSH ключи;
- backup-архивы;
- дампы баз;
- SQLite state DB;
- локальные серверные профили с приватными заметками.

## Базы Данных

UI пытается использовать переменные окружения внутри контейнера базы:

- PostgreSQL: `POSTGRES_USER`;
- MySQL/MariaDB: `MYSQL_USER`, `MYSQL_PASSWORD`, `MYSQL_ROOT_PASSWORD`.

Пароли не выводятся в интерфейсе и не сохраняются в профиле сервера.
