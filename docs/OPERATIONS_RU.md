# Эксплуатация Backup UI

## Анализ сервера

Экран `Анализ` показывает:

- найденные проекты;
- найденные PostgreSQL базы;
- найденные конфигурационные файлы;
- доступность local/SSH хранилищ.

## Создание плана

1. Открыть `Планы`.
2. Нажать `Создать план`.
3. Выбрать проекты.
4. Выбрать базы.
5. Для больших баз включить `пропустить` или сделать отдельный план.
6. Выбрать local и/или SSH хранилище.
7. Настроить retention.
8. Сохранить.

## Запуск backup

Backup запускается кнопкой `Запустить` у плана.

Каждый запуск создает отдельный snapshot-каталог:

```text
/opt/backups/projects/YYYYMMDD-HHMMSS
```

Старые копии не затираются.

## Retention

Экран `Retention` сначала показывает dry-run. Фактическое удаление нужно запускать отдельно, сняв флажок dry-run.

## Восстановление

Экран `Восстановление` позволяет выбрать:

- snapshot;
- компонент;
- режим: файлы или база;
- цель восстановления.

Для запуска нужно ввести:

```text
ВОССТАНОВИТЬ
```

Перед восстановлением остановите сервис проекта и сделайте safety backup текущего состояния.
